What Is This File?
model_portfolio.db
 is a SQLite database pre-loaded with data for a wealth management / portfolio rebalancing problem from the finance industry.

It simulates a scenario where a financial advisor (or robo-advisor platform) manages multiple clients' mutual fund investments and needs to keep their portfolios aligned with a model portfolio — a standardized, target allocation that the firm recommends.

Business Context
Problem: Clients invest in various mutual funds over time. Their actual holdings drift away from the firm's recommended "model portfolio" due to market movements and ad-hoc investments. A portfolio manager needs to identify what to BUY or SELL for each client to bring them back in line.

This is called Portfolio Rebalancing — a core, everyday task in wealth management.

What You Can Build
🥇 Core Feature (Must Have)
Portfolio Rebalancing Engine

For any selected client:

Fetch their holdings and compute current_pct per fund (current_value / total portfolio value)
Compare against model_funds.allocation_pct
Generate:
BUY → fund is underweight vs. model target
SELL → fund is overweight vs. model target
REVIEW → fund not in model (e.g., Axis Bluechip for Amit) — candidate for exit
Save session + items to rebalance_sessions / rebalance_items
Allow advisor to Apply or Dismiss the plan

Rebalancing Logic (Pseudocode)
portfolio_value = SUM(current_value) for client
for each holding:
    current_pct = holding.current_value / portfolio_value * 100
for each model_fund:
    target_value = model_fund.allocation_pct / 100 * portfolio_value
    current_value = client's holding in this fund (0 if not held)
    diff = target_value - current_value
    if diff > 0  → BUY  (amount = diff)
    if diff < 0  → SELL (amount = abs(diff))
for each holding NOT in model_funds:
    → REVIEW / flag for exit
